# VS Code如何行云流水地使用Git
## 分批提交改动
点击“+”把想提交的文件加入stage（暂存区），然后再commit即可。

## 必装的插件——GitLens和Git History二选一
结论先说：**90% 的人直接装 GitLens；只想要“轻量、纯看历史”的简单界面，才选 Git History**。两者不是替代关系，定位完全不一样。

### 一、定位对比（一句话区别）
- **GitLens**：Git 全能增强工具 → **行级溯源+历史+协作+可视化**，深度嵌入编辑器。

- **Git History**：轻量历史浏览器 → **专注提交日志、文件历史、Diff 对比**，界面独立。


### 二、核心功能对比（关键差异）

#### ✅ GitLens（强在“代码上下文”）
1. **行级 Blame（最核心）**  
   每行代码末尾直接显示：**谁、何时、为什么改**，悬停看详情。
2. **CodeLens**  
   文件顶部显示提交次数、最后修改人，一键跳历史。
3. **Commit Graph（提交图谱）**  
   侧边栏可视化分支、合并、提交链，可筛选、搜索、批量操作。
4. **文件/行历史**  
   右键看文件完整历史、某行代码的所有修改记录。
5. **团队协作增强（Pro）**  
   PR 集成、冲突预警、代码审查工具。
6. **高度可定制**  
   可关闭行内提示、调整显示粒度，不干扰 coding。

#### ✅ Git History（强在“简洁独立”）
1. **纯历史视图**  
   独立面板看**全仓库提交日志**，图形化分支链。
2. **文件历史+Diff**  
   右键看文件历史，点击 commit 直接看 Diff，支持多 commit 对比。
3. **搜索过滤**  
   按作者、日期、提交信息、哈希快速筛选。
4. **轻量、低干扰**  
   不嵌入代码区，适合只想查历史、不想界面杂乱的人。

### 三、优缺点对比（直白总结）

#### GitLens
- 优点：**功能最全、上下文最强、团队协作友好、社区免费版足够用**。
- 缺点：**功能多、默认界面信息密集（可关闭）、占用略高**。

#### Git History
- 优点：**轻量、界面干净、专注历史、独立面板不干扰代码**。
- 缺点：**没有行级 Blame、没有 CodeLens、团队协作能力弱**。

### 四、适用人群（直接对号入座）
- 选 **GitLens**，如果你：
  - 想知道**每行代码是谁改的、为什么改**（查 bug、读老代码神器）。
  - 团队协作、经常做 Code Review、处理 PR。
  - 想要**一站式 Git 增强**，不用装多个插件。
- 选 **Git History**，如果你：
  - 个人项目、**只需要看提交历史、对比版本**。
  - 讨厌代码区有额外信息，要**极简干净界面**。
  - 电脑配置一般，想**轻量插件**。

### 五、最终推荐（怎么选最省心）
1. **新手/团队/追求高效**：直接装 **GitLens**（免费版够用），关闭不需要的行内提示，体验最佳。
2. **极简主义/只看历史**：装 **Git History**，搭配 VS Code 内置 Git 即可。
3. **全都想要**：两个可以**同时安装**，互不冲突，按场景切换使用。
